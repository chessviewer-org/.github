# Contributing to chessviewer-org

Thanks for your interest in contributing! This is the organization-wide guide
that applies to every chessviewer-org repository. Most projects also ship their
own `CONTRIBUTING.md` with setup steps and project-specific conventions — when a
repository has one, **that file takes precedence** for anything specific to it.

## Ways to contribute

- **Report a bug** — open an issue with clear steps to reproduce.
- **Request a feature** — describe the problem first, then your idea.
- **Improve docs** — typo fixes and clarifications are always welcome.
- **Submit code** — fix a bug or build a feature. Small, focused pull requests
  are reviewed fastest.

If you're not sure where to start, look for issues labelled **good first issue**
or **help wanted**.

## Before you start

- Search existing issues and pull requests first — someone may already be on it.
- For anything non-trivial, open an issue to discuss the approach before writing
  a lot of code. It saves everyone time.
- By contributing, you agree your work is licensed under the same license as the
  repository you're contributing to.

## Pull requests

- Branch off the repository's default branch.
- Keep each pull request to one logical change.
- Write a clear title and explain **what** changed and **why** — the project's
  pull request template will guide you.
- Make sure the project's checks pass locally before you push.
- Reference the issue your change closes (`Closes #123`).

## Commit messages

We use [Conventional Commits](https://www.conventionalcommits.org/): a short
`type(scope): subject` line — for example `fix(export): handle empty board`.
Keep the subject lower-case and to the point; put the "why" in the body when it
isn't obvious.

## Code of conduct

All participation is governed by our [Code of Conduct](./CODE_OF_CONDUCT.md). Be
kind, be constructive.

## Questions

Open a discussion on the relevant repository, or email
**contact@chessvision.org**.
